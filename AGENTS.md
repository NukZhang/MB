# 仓库指南

## 项目结构
`index.html` 是入口点，连接画布和UI。样式位于 `css/style.css`。游戏逻辑位于 `js/` 目录，按领域组织（`core/`、`entity/`、`component/`、`system/`、`map/`、`weapon/`、`ui/`、`util/`）。游戏玩法数据和调优参数在 `config/*.json` 中，图像资源在 `res/` 中，设计说明保存在 `SPEC/` 中。

## 构建、测试和开发命令
此仓库中没有构建步骤或包管理器。使用任何静态服务器运行游戏或直接打开HTML文件。
- `python -m http.server`（提供仓库根目录，然后打开 `http://localhost:8000`）
- `open index.html`（直接文件加载，用于快速检查）

## 编码风格和命名约定
使用2个空格缩进并保留分号。变量和方法首选 `camelCase`，类和文件使用 `PascalCase`（例如，`js/entity/Player.js`）。每个类放在自己的文件中，并将新模块放在最特定的领域文件夹下。避免添加新的全局变量；如果必须共享状态，请遵循现有的 `window.game` 模式并记录原因。

## 测试指南
当前未配置自动化测试。如果添加测试，请创建 `test/` 目录，文件命名为 `*.test.js`，并更新此文档以包含运行它们的命令。

## 提交和拉取请求指南
Git历史使用简短的单行消息，没有一致的约定。保持提交消息简洁和命令式，并在有帮助时添加简短的范围（例如，`UI: add minimap`）。对于拉取请求，包括简要摘要、手动测试步骤（使用的控件或场景）以及视觉变化的截图或GIF。注意对 `config/` 或 `res/` 的任何更新，以便审查者可以验证数据和资产。
